# Authentifizierung

HelixMind nutzt einen Browser-basierten Login-Flow, um dein CLI mit deinem Web-Plattform-Konto zu verbinden. Nach einmaligem Login werden die Zugangsdaten lokal gecacht und das CLI funktioniert offline.

## Login

```bash
# Browser-basierter Login (empfohlen)
helixmind login

# Direkter API-Key-Login
helixmind login --api-key hm_dein_key_hier

# Eigene Plattform-URL
helixmind login --url https://deine-instanz.example.com

# Erneut authentifizieren (auch wenn bereits eingeloggt)
helixmind login --force
```

### Wie der Browser-Flow funktioniert

1. Das CLI startet einen temporaeren lokalen Server auf `127.0.0.1`
2. Dein Standard-Browser oeffnet die HelixMind-Autorisierungsseite
3. Du loggst dich ein (oder autorisierst sofort, wenn bereits angemeldet)
4. Die Webapp leitet zurueck zum CLI mit einem API-Key
5. Das CLI speichert den Key lokal in `~/.helixmind/config.json`

Der Flow nutzt einen kryptografischen `state`-Parameter als CSRF-Schutz. Der lokale Callback-Server akzeptiert genau eine Antwort und hat ein Timeout von 120 Sekunden.

Falls der Browser nicht automatisch geoeffnet werden kann, gibt das CLI die URL zum manuellen Oeffnen aus.

## Logout

```bash
# Ausloggen und API-Key auf dem Server widerrufen
helixmind logout

# Nur lokal ausloggen (Server-Key bleibt gueltig)
helixmind logout --no-revoke
```

## Status pruefen

```bash
helixmind whoami
```

Zeigt deine E-Mail, Abo-Stufe und maskierten API-Key. Funktioniert offline mit gecachten Daten.

## Slash-Befehle

## Auth Guard — Login vs Open Source

Beim ersten Start von HelixMind siehst du einen **Auswahl-Bildschirm** mit zwei Optionen:

1. **Login** (empfohlen) — kostenloses Konto, schaltet Jarvis AGI, Validation Matrix, Security Monitor und mehr frei
2. **Open Source** — kein Konto noetig, voller CLI-Agent mit 22 Tools und Spiral Memory

HelixMind **erzwingt nie einen Login**. Du kannst immer im Open Source Modus fortfahren. Falls der Login fehlschlaegt, faellt das CLI automatisch in den Open Source Modus zurueck.

Fuehre jederzeit `helixmind login` aus, um von Open Source zum vollen Erlebnis upzugraden.

### Feature-Gating

Im Open Source Modus (ohne Login) sind folgende Befehle gesperrt:

| Gesperrter Befehl | Feature | Freischalten |
|-------------------|---------|-------------|
| `/jarvis` | Jarvis AGI | `helixmind login` (Free+) |
| `/auto` | Autonomer Modus | `helixmind login` (Free+) |
| `/security` | Sicherheits-Audit | `helixmind login` (Free+) |
| `/monitor` | Security Monitor | `helixmind login` (Free+) |
| `/validation` | Validation Matrix | `helixmind login` (Free+) |

Diese Befehle zeigen ein Schloss-Symbol und einen Login-Hinweis wenn sie ohne Konto aufgerufen werden. Sie erscheinen auch nicht in der Autovervollstaendigung.

## API-Keys

API-Keys haben das Praefix `hm_` und werden gehasht (SHA-256) auf dem Server gespeichert. Jeder Key hat Scopes (`cli`, `relay`) und einen Geraetenamen.

Key-Limits nach Plan:

| Plan | Max API-Keys |
|------|-------------|
| Free | 3 |
| Pro | 10 |
| Team | 25 |
| Enterprise | Unbegrenzt |

Du kannst API-Keys ueber dein Web-Dashboard verwalten unter **Dashboard > API Keys**.

## Config-Keys

Die Authentifizierung speichert folgende Keys in `~/.helixmind/config.json`:

| Key | Beschreibung |
|-----|-------------|
| `relay.apiKey` | Dein HelixMind API-Key |
| `relay.url` | Web-Plattform-URL |
| `relay.userId` | Deine User-ID |
| `relay.userEmail` | Deine E-Mail-Adresse |
| `relay.plan` | Deine Abo-Stufe |
| `relay.loginAt` | Zeitstempel des letzten Logins |
| `relay.autoConnect` | Automatisch mit Relay verbinden beim Start |
