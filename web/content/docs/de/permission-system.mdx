# Berechtigungssystem

HelixMind nutzt ein **3-stufiges Berechtigungssystem**. Jeder Tool-Call wird klassifiziert und entweder automatisch ausgefÃ¼hrt, zur Genehmigung vorgelegt oder blockiert.

## Die 3 Stufen

### Safe (Auto-Genehmigung)

Read-only Operationen: `read_file`, `list_directory`, `search_files`, `find_files`, `git_status`, `git_log`, `git_diff`, `spiral_query`, `spiral_store`

### Ask (Nachfragen)

Dateien/State modifizierend: `edit_file`, `write_file`, `git_commit`, `run_command` (bei sicheren Befehlen), `web_research`

### Dangerous (Immer nachfragen)

Potenziell destruktiv: `run_command` mit `rm -rf`, `dd`, `format`, `DROP TABLE`, etc.

## Berechtigungs-Modi

| Modus | Verhalten | Befehl |
|-------|-----------|--------|
| **Standard** | Safe=auto, Ask=prompt, Dangerous=prompt | `helixmind chat` |
| **Skip** | Safe+Ask=auto, Dangerous=prompt | `helixmind -s` / `/skip-permissions` |
| **YOLO** | Alles auto (auch Dangerous!) | `helixmind --yolo` / `/yolo` |
