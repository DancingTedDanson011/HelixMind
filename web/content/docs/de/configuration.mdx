# Konfiguration

HelixMind nutzt ein geschichtetes Konfigurationssystem: globale Config, Projektkontext und Umgebungsvariablen.

## Globale Konfiguration

Gespeichert in `~/.helixmind/config.json`:

```bash
helixmind config set provider anthropic
helixmind config get provider
helixmind config list
```

### Config-Keys

| Key | Beschreibung | Standard |
|-----|-------------|---------|
| `provider` | LLM-Provider | `anthropic` |
| `apiKey` | API-Key (maskiert angezeigt) | — |
| `model` | Modellname | Provider-Standard |
| `logLevel` | Debug-Level | `info` |
| `dataDir` | Datenverzeichnis | `~/.helixmind/` |
| `maxTokens` | Max Kontextfenster | Provider-Standard |
| `validationEnabled` | Validation Matrix | `true` |
| `brainScope` | Brain-Scope | `local` |
| `relay.apiKey` | HelixMind API-Key (gesetzt via `helixmind login`) | — |
| `relay.url` | Web-Plattform-URL | `https://helix-mind.ai` |
| `relay.userId` | User-ID (automatisch beim Login gesetzt) | — |
| `relay.userEmail` | Konto-E-Mail (automatisch beim Login gesetzt) | — |
| `relay.plan` | Abo-Stufe (automatisch beim Login gesetzt) | `FREE` |
| `relay.loginAt` | Zeitstempel des letzten Logins | — |
| `relay.autoConnect` | Automatisch mit Relay verbinden beim Start | `true` |

## Projektkontext (`.helixmind/context.md`)

Wird in **jedes Gespräch** eingebunden. Beschreibe hier Tech-Stack, Architektur, Konventionen und wichtige Regeln deines Projekts.

## Umgebungsvariablen

| Variable | Beschreibung |
|----------|-------------|
| `ANTHROPIC_API_KEY` | Überschreibt Config |
| `OPENAI_API_KEY` | Überschreibt Config |
| `HELIXMIND_DATA_DIR` | Datenverzeichnis überschreiben |
| `HELIXMIND_LOG_LEVEL` | Log-Level überschreiben |

2. **Umgebungsvariablen**

4. **Globale Config** (`~/.helixmind/config.json`)
